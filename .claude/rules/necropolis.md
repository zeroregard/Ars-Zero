# Necropolis Structure Plan

## Overview
Underground jigsaw dungeon accessed via a chainable vertical staircase.
The surface entry point is always at world surface; staircase segments chain
downward until the dungeon bottom lands near Y=30.

---

## ⚠️ What is datagen vs hand-built

### NOT datagen — hand-built in-game, saved via structure block, synced with `./gradlew syncStructures`
All necropolis pieces live in `src/main/resources/data/ars_zero/structure/necropolis/`:
- `entrance_staircase.nbt`
- `hallway_straight.nbt`
- `small_room_1.nbt`
- `medium_room_1.nbt`
- `dead_end_small.nbt`
- `room_connector.nbt` ← **needs to be built and saved in-game**

These are NEVER generated by `./gradlew runData`. Do not add them to `StructureDatagen.java`.

### IS datagen — generated by `./gradlew runData`
Trees, biome features, block tags, worldgen config. No structure NBTs.
`StructureDatagen.java` has been deleted — structure pieces are all hand-built.

---

## Workflow (per piece)

1. Build piece in-game in the Structure Layout world
2. Configure the structure block: name = `ars_zero:necropolis/<piece_name>`
3. Save via the SAVE structure block (button)
4. Run `./gradlew syncStructures` — copies NBT to `src/main/resources/...`, replaces TNT → structure_void
5. Restart client to pick up new NBT
6. Test with `/locate structure ars_zero:necropolis` + tp

**TNT = structure void marker:** place TNT on the outer boundary of a piece while building.
After sync, TNT becomes `minecraft:structure_void` so those positions don't overwrite world terrain on placement.

## Structure JSON
- Type: `ars_zero:necropolis`
- Start pool: `ars_zero:necropolis/start`
- Start jigsaw name: `ars_zero:necropolis/entrance`
- Size: **40** (allows up to ~8 staircase segments)
- Biome tag: `#ars_zero:has_structure/necropolis` → `ars_zero:blight_forest`
- Spawn height: `WORLD_SURFACE_WG` (starts at surface)

## Mob Spawning

Blighted skeletons spawn inside the structure via `spawn_overrides` in `necropolis.json`.
Light level is **NOT checked** — dungeon is decoratively lit; we don't want pitch black required.

### Floor restrictions (`ArsZero.java`)

| Entity | Spawn rule | Floor required |
|--------|-----------|----------------|
| Acolyte | `checkBlightedSpawnRules` | Any `smooth_corrupted_sourcestone*` variant |
| Necromancer | `checkNecromancerSpawnRules` | `smooth_corrupted_sourcestone_small_bricks` only |
| Lich | `checkBlightedSpawnRules` | Any `smooth_corrupted_sourcestone*` variant |

### spawn_overrides in `necropolis.json`
```json
"spawn_overrides": {
  "monster": {
    "bounding_box": "full",
    "spawns": [
      {"type": "ars_zero:acolyte",     "weight": 1, "minCount": 1, "maxCount": 1},
      {"type": "ars_zero:necromancer", "weight": 4, "minCount": 1, "maxCount": 1}
    ]
  }
}
```
`minCount`/`maxCount` = how many per spawn **group attempt**, not a global cap.

## Staircase Chain System

```
World surface (e.g. Y=80)
  [entrance_staircase] — east jigsaw → pool: ars_zero:necropolis/stairs
    weight 3: another entrance_staircase  (chain continues)
    weight 1: room_connector              (dungeon starts)
  [room_connector] — 4 side jigsaws → pool: ars_zero:necropolis/hallways
    → hallway_straight, small_room_1, dead_end_small, small_to_medium ...
```

Jigsaw names used by staircase:
- `name`: `ars_zero:necropolis/entrance`
- `target`: `ars_zero:necropolis/connector`

## Template Pools

| Pool | File | Purpose |
|------|------|---------|
| `necropolis/start` | `start.json` | Start piece → entrance_staircase |
| `necropolis/stairs` | `stairs.json` | Chain: staircase (w3) or room_connector (w1) |
| `necropolis/hallways` | `hallways.json` | hallway_straight, small_room_1, dead_end_small, medium_room_1 |
